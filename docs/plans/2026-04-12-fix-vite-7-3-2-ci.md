# Fix Vite 7.3.2 CI Failures

## Goal

Stabilize the Dependabot branch that upgrades Vite to 7.3.2 and restore CI test execution.

## Context

- CI on the Dependabot branch fails during Vitest execution with `Unknown method: getBuiltins. Expected "fetchModule".`
- Local reproduction on Windows does not match CI exactly, but the failing branch still shows test-runner instability under `pnpm nx affected --target=test`.
- The error string is emitted from `vitest@4.0.0` module-runner transport code, which suggests a Vitest/Vite protocol mismatch rather than an application-code regression.

## Plan

1. Upgrade the Vitest package set to a newer 4.x release compatible with Vite 7.3.2.
2. Reinstall dependencies to refresh `pnpm-lock.yaml`.
3. Re-run affected test targets, prioritizing the targets called out in CI.
4. Summarize the root cause, evidence, and any remaining issues.

## Validation

- `pnpm nx affected --target=test --outputStyle=static`
- Focused reruns for any remaining failing projects
