# Archived Plans

This folder contains historical planning and review documents that are no longer meant to guide current implementation work.

What belongs here:

- Completed implementation plans that are useful mainly as historical record
- Point-in-time review notes tied to a specific branch, audit, or milestone
- Planning artifacts whose "current state", task lists, or branch assumptions are no longer current

What should stay in `docs/plans/` instead:

- Active planning guidance for the repository
- Requirements or scope documents that still define current expectations
- Completed plans that remain useful as active reference material

Archived documents may contain:

- stale branch names
- outdated inventories
- superseded next steps
- findings that were later resolved

Keep archived files readable, but do not rewrite them into current-source-of-truth documentation.
