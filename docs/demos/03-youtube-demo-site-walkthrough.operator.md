# Recording Operator Script: The Sign Age YouTube Walkthrough

Use this version when driving the demo with Chrome DevTools MCP.

**Target runtime:** 3-4 minutes

## Before Recording

- make sure the entire browser window is visible in frame
- confirm browser zoom is where you want it
- hide unrelated tabs, bookmarks, and notifications
- start with a clean browser state if possible
- keep mouse movement minimal and let the narration lead the navigation

## Chrome DevTools MCP Notes

- use direct URL navigation rather than scrolling between sections
- let the pre-roll line start just before the navigation action
- after each navigation, hold the page long enough for the viewer to orient before continuing
- keep the GitHub portion short and proof-oriented

## Sequence

1. Start URL: <https://cambridgemonorail.github.io/TheSignAge/#/signage/welcome>
   Hold: 10-12s
   Line: "Web developers can build almost anything. Apps. Dashboards. Entire products. But there is one place software still feels strangely difficult to build for. Walls. Offices. Shops. Hospitals. Stations. Screens everywhere. Yet most of them are still powered by slide decks and CMS templates. So this project asks a simple question. What if building digital signage felt like normal frontend development?"

2. Stay on: <https://cambridgemonorail.github.io/TheSignAge/#/signage/welcome>
   Hold: 6-8s
   Line: "If you can build a React interface, you should be able to build a screen like this. Software that does not just live in browsers. Software that lives on walls."

3. Navigate to: <https://cambridgemonorail.github.io/TheSignAge/#/signage/welcome>
   Pre-roll: "Start with the output."
   Hold: 8-10s
   Line: "A screen designed for distance. Large type. Clear hierarchy. Information that can be understood instantly from across a room. This is the design constraint that makes signage different from web UI."

4. Navigate to: <https://cambridgemonorail.github.io/TheSignAge/>
   Pre-roll: "The Sign Age treats digital signage as software."
   Hold: 12-15s
   Line: "Instead of slide decks, we build screens using components and layout systems. The same ideas frontend developers already understand. Deterministic layout. Installable UI. Real code."

5. Navigate to: <https://cambridgemonorail.github.io/TheSignAge/#/getting-started>
   Pre-roll: "The goal is to make signage development feel familiar."
   Hold: 12-15s
   Line: "Clone the repository. Install components from the registry. Or copy the source directly into an existing React project. No proprietary editors. No mysterious tooling. Just code."

6. Navigate to: <https://cambridgemonorail.github.io/TheSignAge/#/components>
   Pre-roll: "Like most UI systems, it starts with primitives."
   Hold: 8-10s
   Line: "Small building blocks. Layout systems. Then higher level patterns built on top. Familiar territory for any frontend developer."

7. Navigate to: <https://cambridgemonorail.github.io/TheSignAge/#/components/primitives/metric-card>
   Pre-roll: "Here is a simple primitive."
   Hold: 10-12s
   Line: "A metric card designed specifically for large displays. Strong contrast. Big typography. Readable from distance. The constraints of signage drive the design."

8. Navigate to: <https://cambridgemonorail.github.io/TheSignAge/#/components/layouts/split-screen>
   Pre-roll: "Layouts are where signage becomes interesting."
   Hold: 10-12s
   Line: "Screens usually have fixed zones. Rather than responsive chaos, we use deterministic layouts. You know exactly where content will appear."

9. Navigate to: <https://cambridgemonorail.github.io/TheSignAge/#/gallery>
   Pre-roll: "Once you have primitives and layouts, the patterns start to emerge."
   Hold: 8-10s
   Line: "The gallery shows complete screen examples built from those pieces. Different use cases. Same component system."

10. Navigate to: <https://cambridgemonorail.github.io/TheSignAge/#/signage/menu>
    Pre-roll: "Menu boards are a classic signage challenge."
    Hold: 8-10s
    Line: "Lots of information. Limited space. Structure and typography keep the screen readable."

11. Navigate to: <https://cambridgemonorail.github.io/TheSignAge/#/signage/announcements>
    Pre-roll: "Another common pattern is internal communication."
    Hold: 8-10s

12. Navigate to: <https://github.com/CambridgeMonorail/TheSignAge?tab=readme-ov-file#the-sign-age>
    Pre-roll: "Behind the demo is a real repository."
    Hold: 6-8s
    Line: "Libraries. Documentation. Tooling. Everything needed to build screens like these."

13. Navigate to: <https://github.com/CambridgeMonorail/TheSignAge/tree/main/skills>
    Hold: 6-8s

    Hold: 6-8s

    Hold: 10-12s
    Line: "The ambition here is quite simple. If you can build a modern web interface, you should be able to build a screen like this. Software that lives on walls."

16. Optional end card
    Hold: 4-6s
    Line: "If this idea interests you, the project is open source and still evolving. Links are in the description."

## Operator Notes

- keep the first 10 seconds clean and deliberate because that section carries the hook
- do not scroll unless a page fails to land in a useful position after navigation
- if pacing runs long, trim hold time on the component index and GitHub stops first
- if a page loads slowly, wait for stability before speaking the main line
